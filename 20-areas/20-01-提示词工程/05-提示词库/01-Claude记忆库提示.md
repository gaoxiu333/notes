# Claude记忆库提示词

## 记忆库基础

Claude内存库允许AI助手记住跨会话的重要信息，有效地个性化交互和建立长期"关系"。本文档提供了优化使用记忆库功能的核心提示词模板。

## 内存创建提示词

### 基础内存创建

```
请将以下信息添加到你的记忆库：

主题：[信息主题]
重要级别：[高/中/低]

内容：
[详细描述需要记住的信息]

使用场景：
- 当我询问[相关主题]时
- 当对话涉及[相关概念]时
- 用于[特定任务]时

这些信息应该[永久保存/保存到特定日期]。
```

### 组织化内存创建

```
请将以下结构化信息添加到你的记忆库：

类别：[主要分类]
子类别：[次要分类]
优先级：[1-5，5最高]

核心内容：
- 关键点1：[详细描述]
- 关键点2：[详细描述]
- 关键点3：[详细描述]

相关联系：
- 与[其他已存储信息]的关系：[描述关联]
- 适用条件：[何时应使用此信息]
- 排除条件：[何时不应使用此信息]

检索标签：[标签1], [标签2], [标签3]

请确认存储完成，并描述你将如何在未来对话中使用这些信息。
```

### 个人偏好记忆模板

```
请记住我的以下个人偏好：

沟通风格偏好：
- 我喜欢[简洁/详细/幽默/严肃]的回答
- 我偏好[直接给出结论/先给出推理过程]
- 我[需要/不需要]大量例子来理解概念

专业领域偏好：
- 我的专业背景是[专业领域]
- 我熟悉[技术/概念/工具]
- 我不熟悉[技术/概念/工具]，需要简化解释

特定需求：
- 当讨论[主题]时，请始终考虑[特定因素]
- 我对[特定信息]特别感兴趣，请优先提供
- 当提到[触发词]时，请提醒我[相关事项]

请记住这些偏好并在所有未来交互中应用，使我们的对话更加高效。
```

## 记忆检索与使用提示词

### 定向记忆检索

```
请从你的记忆库中检索关于[主题/概念]的信息：

具体我需要了解：
- [具体查询点1]
- [具体查询点2]
- 我们之前讨论过的关于[特定细节]

如果存在多条相关记忆，请：
1. 按相关性排序呈现
2. 说明每条记忆的来源和添加时间
3. 区分确认过的事实和推测性信息

在提供记忆内容后，请解释这些信息如何应用于当前讨论的上下文。
```

### 记忆应用与整合

```
请将你记忆库中的信息应用到以下问题：

[具体问题或任务]

在回答时：
1. 明确指出哪些部分基于记忆库中的信息
2. 整合我们过去讨论中的具体例子或见解
3. 考虑我的偏好、背景和已表达的观点
4. 注明哪些部分是你的新分析而非记忆内容

请确保记忆内容和新信息之间的无缝整合，使回答个性化但保持准确和有用。
```

### 记忆对比与更新

```
请比较以下新信息与你记忆库中的已有信息：

新信息：
[详细描述新信息]

请执行以下操作：
1. 检索记忆库中与此主题相关的现有信息
2. 指出新旧信息之间的一致点和差异点
3. 分析差异可能的原因（信息更新、观点变化等）
4. 建议如何更新记忆库以整合新信息

对于有冲突的信息，请推荐哪个版本应该保留或两者如何合并，并解释原因。
```

## 记忆管理提示词

### 记忆更新与修正

```
请更新你记忆库中关于[主题]的信息：

当前存储的信息：[简要描述目前记忆的内容]

需要更新的内容：
- 更正：[需要修正的错误信息]
- 添加：[需要补充的新信息]
- 删除：[需要移除的过时或不准确信息]
- 重新优先级：将此信息优先级调整为[新优先级]

更新原因：[说明为什么需要这次更新]

请确认更新完成，并回顾更新后的完整信息，以确保准确性。
```

### 记忆整理与分类

```
请帮我整理你记忆库中关于[广泛主题]的信息：

组织请求：
1. 将相关记忆分类为以下类别：
   - [类别1]：相关记忆及简要描述
   - [类别2]：相关记忆及简要描述
   - [类别3]：相关记忆及简要描述

2. 识别重复或冗余信息，建议如何合并

3. 指出信息缺口：根据现有记忆，哪些重要信息还缺失？

4. 创建主要概念的关系图：说明不同记忆点之间如何相互关联

完成整理后，请提供一个记忆"目录"，概述主要类别和关键点，便于未来检索。
```

### 记忆审查与清理

```
请审查你记忆库中的信息，并推荐清理策略：

审查范围：
- 时间范围：[时间段]的记忆
- 主题范围：关于[主题]的记忆
- 优先级范围：优先级[低于等级]的记忆

审查标准：
- 相关性：对我当前项目和兴趣的相关程度
- 时效性：信息是否过时或被新信息取代
- 准确性：信息是否已被证实为不准确
- 使用频率：我们多久引用一次这些信息

请提供：
1. 建议保留的高价值记忆摘要
2. 建议归档或删除的低价值记忆列表
3. 建议更新或验证的可能过时记忆

在提出建议后，请等待我的确认再执行任何记忆修改操作。
```

## 高级记忆应用

### 个性化对话框架

```
请使用你的记忆库创建一个个性化的对话框架：

基于你记住的我的：
- 沟通偏好和风格
- 专业背景和知识领域
- 典型问题和关注点
- 过去的重要对话和决定

创建一个框架，定义：
1. 你如何结构化回答以最适合我的需求
2. 何时主动提及记忆库中的相关信息
3. 如何在不重复已知信息的情况下建立连贯性
4. 如何预测我可能的后续问题和关注点

请说明这个框架如何用于使我们的对话更高效和个性化，并举例说明它将如何应用于几种典型交互场景。
```

### 知识演化追踪

```
请追踪我们对[主题/概念]理解的演化：

使用你的记忆库：
1. 识别我们最早讨论此主题的时间和初始理解
2. 追踪关键的理解转变点和新见解
3. 比较最初与当前理解的主要差异
4. 分析导致理解变化的因素（新信息、新视角等）

请以时间线形式展示这一演化过程，包括：
- 关键日期和对话
- 每阶段的核心概念和假设
- 重要的转变点和催化剂
- 我们可能仍存在的知识缺口

这一分析将帮助我了解我的学习历程，并识别下一步应探索的方向。
```

### 情境激活规则设置

```
请为你的记忆库设置以下情境激活规则：

规则1 - 主题触发：
- 当讨论[特定主题]时，自动提取并应用相关记忆
- 优先级：[高/中/低]
- 应用方式：[主动提及/仅在相关时/仅在被问及时]

规则2 - 时间触发：
- 在[特定日期/周期]，提醒我关于[特定事项]
- 提醒内容：[简要描述]
- 提醒形式：[简单提示/详细回顾]

规则3 - 关联触发：
- 当提到[概念A]时，联想并提供关于相关[概念B]的记忆
- 关联强度：[强（总是联想）/中/弱（仅当高度相关时）]
- 关联理由：[解释为何这些概念相关]

规则4 - 条件触发：
- 仅当[特定条件]满足时，才提供关于[主题]的记忆
- 条件详情：[详细描述触发条件]
- 排除条件：[何时不应触发]

请确认这些规则已设置，并解释它们将如何影响我们未来的交互。
```
