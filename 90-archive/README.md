## 归档原则

- 已完成的项目：完成所有交付物且无需后续维护
- 弃置的项目：决定不再继续的项目
- 过时的资源：已经不再适用的技术资料
- 废弃的笔记：内容已整合到其他笔记或不再需要

## 归档流程

1. 确认项目/笔记完成状态，更新其元数据
2. 将项目/笔记从原位置移动到对应年份的归档文件夹
3. 确保所有指向该内容的链接已经更新
4. 在原项目/笔记的位置留下归档说明（可选）

## 归档内容查询

使用Dataview查询特定归档内容：

```dataview
LIST
FROM "90-archive"
WHERE contains(file.path, "项目名称")
```

## 归档内容恢复

如需恢复归档内容：

1. 复制归档内容到相应的活动目录
2. 更新状态和日期元数据
3. 检查并更新链接关系
